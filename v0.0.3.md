# PRD: Slate OS v0.0.3 — Core Desktop UI

**Version:** v0.0.3  
**Status:** Draft  
**Author:** Adam  
**Last Updated:** 2026-07-03

---

# Overview

Slate OS v0.0.3 introduces the first version of the Slate Desktop Environment.

Previous releases established a stable graphical foundation and visual identity. This milestone introduces the first native desktop interface while keeping functionality intentionally minimal.

The objective is to replace the appearance of a generic Hyprland session with a desktop that behaves like Slate.

---

# Objective

Create the first usable desktop interface.

Users should be able to interact with the operating system without relying primarily on keyboard shortcuts.

---

# Problem Statement

Although Slate now has its own branding, interaction still depends on Hyprland defaults.

Users have no consistent navigation model.

There is no permanent desktop interface.

This milestone introduces the core UI components that will become the foundation for future releases.

# Functional Requirements

## FR-001 — Top Bar

A persistent top bar shall be displayed.

The top bar shall remain visible during normal desktop usage.

---

## FR-002 — Live Clock

Display current system time.

Time shall update automatically.

No calendar is required.

---

## FR-003 — System Status

Display basic status indicators.

Minimum:

- battery (if available)
- audio
- network

Interaction is not required.

Indicators only need to display state.

---

## FR-004 — Bottom Dock

Display a persistent application dock.

The dock shall remain visible.

---

## FR-005 — Application Launch

Pinned applications shall launch when clicked.

Initially supported:

- Terminal

Future applications are out of scope.

---

## FR-006 — Active Application Indicator

The dock shall visually indicate running applications.

---

## FR-007 — Desktop Focus

Clicking the wallpaper removes focus from windows.

---

# Non-Functional Requirements

## Performance

Desktop interactions should feel immediate.

No noticeable lag should occur during normal use.

---

## Stability

The UI must not introduce crashes.

---

## Simplicity

Avoid adding advanced features.

Every component should have a single responsibility.

---

# Out of Scope

The following remain excluded:

- Login screen
- Notifications
- Search
- Application launcher
- Widgets
- Weather
- Calendar popup
- Control Center
- Bluetooth controls
- Wi-Fi settings
- Clipboard history
- Virtual desktops UI
- Mission Control
- Window snapping
- Custom animations
- Settings application
- Slate Store

---

# Design Principles

The interface should follow these principles:

- minimal
- distraction-free
- consistent spacing
- clear hierarchy
- predictable behavior

The desktop should never feel visually cluttered.

---

# Success Criteria

The milestone is complete when:

- Top bar is displayed.
- Clock updates correctly.
- Status indicators render correctly.
- Dock is displayed.
- Terminal launches from the dock.
- Running applications are indicated.
- Existing functionality from previous releases remains stable.

---

# Failure Criteria

The milestone fails if:

- Top bar disappears unexpectedly.
- Dock becomes unusable.
- Applications fail to launch.
- UI freezes.
- Existing desktop functionality regresses.

---

# Validation Procedure

1. Boot Slate OS.
2. Verify top bar rendering.
3. Verify live clock.
4. Verify status indicators.
5. Launch Terminal from the dock.
6. Close Terminal.
7. Launch Terminal again.
8. Repeat multiple times.
9. Confirm no regressions from v0.0.2.

---

# Risks

| Risk | Mitigation |
|------|------------|
| Dock becomes too feature-rich | Limit to launching applications only |
| Performance degradation | Keep rendering lightweight |
| UI inconsistency | Follow the Slate design system |
| Feature creep | Exclude launcher and notifications |

---

# Deliverables

At the end of this milestone the project provides:

- Slate Top Bar
- Live Clock
- System Status Indicators
- Slate Dock
- Dock-based application launching
- Running application indicators

The operating system now includes its first native desktop interface.
