# PRD: Slate OS v0.0.4 — Application Launcher & Window Management

**Version:** v0.0.4  
**Status:** Draft  
**Author:** Adam  
**Last Updated:** 2026-07-03

---

# Overview

Slate OS v0.0.4 introduces the first complete application workflow.

Previous milestones established a stable graphical foundation, branding and the core desktop interface.

This release enables users to discover, launch and switch between installed applications without relying on the terminal or keyboard shortcuts.

---

# Objective

Provide the first complete application experience.

Users should be able to:

- browse installed applications
- launch applications
- switch between running applications
- manage application windows

---

# Problem Statement

Current versions require users to know keyboard shortcuts or manually launch applications.

A desktop environment without an application launcher is incomplete.

This milestone establishes the primary interaction model for opening applications.

---

# Users

Primary user:

- Adam (developer)

# Goals

- Introduce the Slate Launcher.
- Improve window management.
- Reduce dependency on Hyprland shortcuts.

---

# Functional Requirements

## FR-001 — Slate Launcher

The system shall include a native application launcher.

The launcher shall display all installed graphical applications.

---

## FR-002 — Launcher Activation

The launcher shall open by:

- clicking the Apps button in the dock

Keyboard shortcuts are optional.

---

## FR-003 — Application Grid

Applications shall be displayed in a grid layout.

Each application shall display:

- icon
- name

Folders are out of scope.

---

## FR-004 — Launch Applications

Selecting an application launches it.

The launcher closes automatically after launch.

---

## FR-005 — Running Application Indicators

Running applications shall be visually distinguished.

---

## FR-006 — Window Focus

Selecting a running application shall focus its window.

---

## FR-007 — Window Controls

Application windows shall support:

- Close
- Minimize
- Maximize / Restore

---

## FR-008 — Multiple Windows

Multiple application windows shall be supported simultaneously.

---

## FR-009 — Basic Window Animations

Opening and closing windows shall use lightweight animations.

Animation polish is out of scope.

---

# Non-Functional Requirements

## Performance

Launcher should open within approximately 300 ms.

---

## Stability

Launcher shall not crash during repeated opening and closing.

---

## Simplicity

Launcher should prioritize clarity over advanced functionality.

---

# Out of Scope

The following remain excluded:

- Search
- Categories
- Folders
- Favorites management
- Widgets
- Notifications
- Settings
- Login screen
- User accounts
- Slate Store
- Drag-and-drop organization
- Context menus
- Multi-monitor customization

---

# Design Principles

The launcher should be:

- minimal
- keyboard-friendly
- visually consistent
- distraction-free

Users should understand how to launch an application immediately.

---

# Success Criteria

The milestone is complete when:

- Launcher opens correctly.
- Installed applications are displayed.
- Applications launch successfully.
- Launcher closes automatically after launch.
- Running applications are indicated.
- Window controls function correctly.
- Multiple windows work reliably.

---

# Failure Criteria

The milestone fails if:

- Launcher fails to open.
- Applications cannot launch.
- Window controls fail.
- Launcher crashes.
- Existing desktop functionality regresses.

---

# Validation Procedure

1. Boot Slate OS.
2. Open the launcher.
3. Launch Terminal.
4. Launch a second application.
5. Switch between applications.
6. Minimize a window.
7. Restore it.
8. Maximize a window.
9. Close all applications.
10. Repeat multiple times.

---

# Risks

| Risk | Mitigation |
|------|------------|
| Launcher becomes overly complex | Focus on launching only |
| Window management bugs | Keep Hyprland handling window logic where possible |
| Performance degradation | Render launcher only when opened |
| Scope creep | Exclude search and categories |

---

# Deliverables

At the end of this milestone the project provides:

- Slate Launcher
- Application Grid
- Application Launching
- Running Application Indicators
- Full Window Controls
- Basic Window Animations

Slate OS can now be operated without relying on terminal commands or compositor shortcuts.
